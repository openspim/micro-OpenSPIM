SELECT CHANNELS/PINS

This is the place to add or activate one or more channels such as various laser lines or other hardware components of the OpenSPIM system. There are two ways to do this: Software controlled (without an ArduinoUNO microcontroller) and Arduino Controlled (with an ArduinoUNO microcontroller).

Software controlled versus hardware controlled image acquisition

“Software controlled” indicates that during image acquisition the software alone controls all hardware components of the microscope, such as camera exposure times and laser shutters. In this case images cannot be acquired as fast as in microscopes where some or most of the hardware components are controlled via triggering signals coming e.g. from an ArduinoUNO microcontroller, i.e. “hardware controlled”. However, if the intention is to capture slow developmental processes with acquisition speed playing a subordinate role, software-controlled acquisition should be sufficient.

An ArduinoUno board allows to speed up acquisition speed by connecting the board’s pins to certain hardware components of the OpenSPIM system. As shown in the figure below one can e.g. connect two of the Arduino pins directly to two separate laser lines whereby the Arduino shutter takes over and triggers the respective lasers. As a result, the lasers will fire faster benefitting acquisition speed. It is important to note that the OpenSPIM plugin assumes that the camera will be connected to the Arduino board and therefore provides the initial 5V TTL-trigger-pulse.

Set up a “Software Controlled” imaging session

Select the “Software Controlled” tab within the “Select Channels/Pins” panel. Click on “Add a channel” to add one or several laser lines. Changes can also be made in the drop-down menus. The exposure time for each channel during the acquisition is also set here.

Set up an “Arduino controlled” imaging session

Select the “Arduino Controlled” tab within the “Select Channels/Pins” panel. In order to initially access this tab, the ArduinoUNO has to be configured at least once in the OpenSPIM Configurations menu, located on the very left of the GUI; double click on ArduinoUno and - if necessary - follow the help guide. Save the plugin to skip this step next time you start the OpenSPIM pugin.

In order to trigger the six individual output pins of the ArduinoUNO board (Pin 9-13). Every hardware component that is connected to and triggered by these pins, such as lasers or other light sources, can be considered also as channels. Channels of each corresponding pin can be added to the acquisition simply by ticking the individual checkboxes. Tick the “simultaneous triggering” box to trigger the selected channels at the same time. Note that the moment you go into LIVE mode, all ticked channels/pins will already provide the 5V-trigger-pulse. If a laser is already connected and correctly set to receive the trigger-pulse it will now fire in accordance with the current exposure value of the camera. However, for the image acquisition itself only the exposure time specified next to the Channel name will be considered.
